# Setup — getting this profile live

A friendly, copy-paste guide. You do **not** need to be a developer. There's a
terminal step or two; each command says what it does. (Dev details live in
`BUILD.md` / `PROFILE-SPEC.md`.)

---

## 1. Create the special repo

GitHub shows a repo's `README.md` on your profile **only if the repo name exactly
matches your username**. So:

1. On GitHub, click **New repository**.
2. Repository name: **`frankiehope333-ctrl`** (must exactly equal your username).
3. Make it **Public**, and **do not** add a README/license (this project has one).
4. Click **Create repository**.

> Already wired to username `frankiehope333-ctrl` in `config.py` and the dashboard
> cards. If you ever change usernames, update `GH_USER` in `config.py` and the
> `username=` values in `sections/05-dashboard.md`.

---

## 2. Push these files to it

In a terminal, from the folder that contains this `SETUP.md`:

```bash
git init                         # start tracking this folder with git
git add .                        # stage every file
git commit -m "🌊 CLEAVE profile — initial build"
git branch -M main               # name the branch 'main'
git remote add origin https://github.com/frankiehope333-ctrl/frankiehope333-ctrl.git
git push -u origin main          # upload it
```

Refresh your GitHub profile — the page is now live. 🎉

---

## 3. Turn on the games (one click)

The reef and the visitor games run on **GitHub Actions** (free automation).

1. Go to your repo → **Settings** → **Actions** → **General**.
2. Under **Workflow permissions**, choose **Read and write permissions**, Save.
   *(This lets the daily "tide" commit the updated reef back to the repo.)*
3. Go to the **Actions** tab → if prompted, click **enable workflows**.

That's it. From now on:
- **🌊 tides** runs every day at 6am UTC and grows your reef from your commits.
  (You can also run it by hand: Actions → *tides* → **Run workflow**.)
- **🎮 on-issue** runs whenever a visitor uses a bottle/creature link.

## 4. Fill in the blanks (search for `TODO: confirm`)

A few spots are intentionally left for you so nothing fake ships:

| Where | What to add |
|---|---|
| `sections/02-links.md` | Real URLs for website / LinkedIn / TikTok / Instagram / email |
| `sections/05-dashboard.md` | Your **real** impact numbers (clients, hours saved…), then uncomment that block. *Don't ship made-up numbers.* |
| `sections/03-building.md` | Repo/one-pager links for each project |
| `sections/adventure/ending-*.md` | Point the `[CLEAVE](#)` link at `cleave.consulting` |
| `sections/01-about.md` | Double-check the certs + company names are right |

After editing any `sections/*.md`, regenerate the README:

```bash
pip install -r requirements.txt   # one-time: install Pillow + requests
python build_readme.py            # rewrites README.md from the sections
git commit -am "update copy" && git push
```

> Never edit `README.md` by hand — it's generated. Edit the `sections/` files.

## 5. Moderation: bottles start in "manual" mode (safe)

Visitor messages are held until you approve them, so nothing unexpected lands on
your profile. When someone sends a bottle:
- An **issue** appears in your repo, auto-labeled `pending`.
- Read it. If you're happy, add the **`approved`** label → it publishes itself.
- If not, just close the issue (or delete it).

Once you trust it (a couple of weeks), flip to automatic: in `config.py` set
`BOTTLE_MODERATION = "auto"`, commit, push. The blocklist filter still applies.

## 6. The art (🎨 drops in later)

Five illustrations come from Higgsfield and just get dropped into `assets/`:
`banner.png`, `marlowe.png`, `cass.png`, and adventure backgrounds in
`assets/adventure/`. Until they exist, the profile uses tasteful text/gradient
placeholders, so it never looks broken. When `banner.png` is ready, open
`sections/00-banner.md` and follow the one-line note at the top to swap it in.

## Questions?

The little pixel GIFs (reef, bottle, creature) are generated by Python in
`render/`. You can preview any of them locally:

```bash
python render/reef_gif.py        # writes render/reef.gif
python render/creature_gif.py    # writes render/creature.gif
python render/bottle_gif.py      # writes render/bottle_bob.gif
```
